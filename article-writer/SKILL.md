# 自媒体技术文章写作助手

支持将技术内容适配到知乎、CSDN、微信公众号、小红书四个平台。

## 工作流程

1. **收集素材** — 用户提供主题/大纲/代码/仓库链接
2. **确定平台** — 用户指定目标平台（可选多平台）
3. **读取平台规范** — 根据平台读取对应的 `references/<platform>.md`
4. **生成文章** — 按平台风格规范生成内容
5. **技术元素处理** — 按规范处理代码块、仓库链接、架构图等技术元素

## 平台快速识别

| 平台 | 文件 | 适用场景 |
|------|------|----------|
| 知乎 | `references/zhihu.md` | 深度技术解析、原理剖析、经验分享 |
| CSDN | `references/csdn.md` | 教程、踩坑记录、安装配置、解决方案 |
| 公众号 | `references/wechat.md` | 技术传播、品牌建设、粉丝互动 |
| 小红书 | `references/xiaohongshu.md` | 工具推荐、学习笔记、轻量技巧 |

## 技术内容处理规则

### GitHub 仓库链接

- **知乎/CSDN**：直接内嵌链接，必要时附上仓库 README 核心内容摘要
- **公众号**：转为文字描述 + 二维码提示（"扫码查看仓库"），或截图展示
- **小红书**：不直接贴链接（平台限制），改为"搜索 GitHub 用户名/仓库名"

### 代码片段

- **知乎/CSDN**：完整代码块 + 逐行/逐段注释 + 运行结果展示
- **公众号**：代码截图为主（公众号代码渲染差），关键代码可内联展示
- **小红书**：不贴代码，改用"代码截图"或"伪代码描述"，聚焦思路而非实现

### 技术架构/流程

- **知乎/CSDN**：文字描述 + Mermaid/ASCII 图 + 表格对比
- **公众号**：精心设计的架构图截图 + 简洁文字说明
- **小红书**：简化为流程卡片图 + 一句话概括每步

## 多平台一键生成

当用户要求"多平台版本"时，依次读取所需平台的 references 文件，生成所有版本。每个版本以 `---` 分隔，开头标注平台名。

## 用户指令示例

- `写一篇关于 xxx 的知乎文章`
- `把这篇内容改成 CSDN 风格`
- `生成小红书版本的技术笔记`
- `帮我写公众号技术推文，主题是 xxx`
- `为这个仓库写一篇多平台推广文章 [仓库链接]`

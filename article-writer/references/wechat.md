## 平台定位

公众号是**封闭生态**，依赖社交转发和粉丝传播。标题决定打开率，内容决定转发率。技术类公众号需在专业性和可读性之间取得平衡，读者期待「通俗易懂但又不失深度」的内容。

## 标题风格

- **标题是生命线**：决定了 80% 的打开率
- 常见高打开率标题模式：
  - 数字式：「5 个技巧让你的 API 性能提升 10 倍」
  - 悬念式：「这个 Go 并发问题，90% 的人都会踩坑」
  - 对比式：「为什么我放弃了 Spring Boot 转向 Quarkus」
  - 痛点式：「线上 OOM 了？别慌，教你 3 步定位」
  - 热点式：「DeepSeek 开源背后的技术架构解析」
- 标题长度 15-30 字为佳
- 适当制造紧迫感和好奇心

## 文章结构

### 技术推文结构

```
封面图（精心设计，品牌感）

开头钩子（3行以内）
  → 一句话讲清楚「看完能得到什么」
  → 制造阅读动机

正文（1500-3000字）
  → 短段落，快节奏
  → 每 300 字一个视觉锚点（图/代码/引用/分割线）

结尾
  → 总结核心价值
  → 引导关注/转发/点赞
  → 「在看」引导语
```

## 语言风格

- **通俗易懂**是核心：把复杂技术用人话讲清楚
- 可以活泼但不能浮夸，保持专业度
- 多用类比和生活化比喻
- 站在读者角度写：「你可能会遇到……」「这里有个坑……」
- 适当加入个人风格和观点
- 语气像资深同事在茶水间分享经验

## 排版规范

### 基础规范
- **正文 15px 字号**，行间距 1.75，字间距 1
- 段落间距适当，善用空行
- **不原生支持 Markdown**，需借助排版工具
- 每段不超过 4-5 行（手机阅读友好）
- 重点内容用颜色标注（品牌色或红色）

### 视觉要求
- **封面图**：900x383px，精心设计，有品牌感
- 代码截图优于直接贴代码（公众号代码渲染差）
- 适当使用 GIF 动图展示操作流程
- 文章配图风格统一

### 排版工具推荐
用户可使用以下工具进行排版：
- Markdown Nice / md2wechat：Markdown 转公众号排版
- 壹伴助手：浏览器插件，AI 排版
- 135 编辑器：专业排版工具

## 技术元素处理

### 代码
- **截图为主**：使用 Carbon 之类的代码截图工具
- 关键代码可以内联展示，但不宜超过 10 行
- 长代码引导至 GitHub 或 Gist 查看
- 配置文件展示关键部分即可

### 仓库链接
- 公众号内链接不可点击（仅服务号底部可放链接）
- 处理方式：
  - 「阅读原文」链接指向仓库
  - 文中描述 + 引导点击原文
  - 二维码图片（「扫码查看源码」）
  - 文字形式：「GitHub 搜索 xxx/yyy」

### 架构图/流程图
- 精心绘制的架构图截图
- 用颜色区分不同模块
- 配合简洁的文字说明
- 推荐工具：Draw.io、Excalidraw

## 运营技巧

1. **开头 3 行定生死**：直接告诉读者"你能得到什么"
2. **节奏感**：文字 → 图 → 文字 → 代码，交替出现
3. **系列化内容**：建立专栏意识，持续输出形成品牌
4. **固定栏目**：如「每周一技」「踩坑记录」「源码解析」
5. **互动引导**：结尾必须有 CTA（Call to Action）
6. **发布时间**：工作日早 8-9 点、午 12-13 点、晚 20-22 点

```markdown
{内容，短段落}

{配图或代码截图}

## {小标题2}

{内容}

{3-5 个要点总结}

---

{引导语}

- 关注「{公众号名}」，获取更多技术干货
- 有问题？评论区见👇
```

## 平台定位

CSDN 是**国内最大的开发者社区**，读者主要通过搜索引擎进入，目标是快速找到解决方案。内容偏实操性、工具性，以教程、配置指南、踩坑记录为主。文章长度 1500-5000 字为宜。

## 标题风格

- **关键词优先**：标题要包含核心技术关键词，利于 SEO
- 格式常见：「xxx 的安装与配置」「xxx 报错解决方案」「使用 xxx 实现 yyy」
- 适当使用数字和年份增强时效性：「2025 年最新」「5 步搞定」
- 避免过于泛化的标题，要具体明确
- 好标题示例：
  - 「Spring Boot 3.2 整合 MyBatis-Plus 完整教程（附源码）」
  - 「Docker 容器网络不通？5 种排查方案一次讲清」
  - 「Go 语言 Gin 框架中间件原理与自定义实战」

## 文章结构

### 标准教程结构

```
环境说明（简洁列出版本/系统）
  → 前置条件/依赖

步骤1：安装/准备
步骤2：核心配置
步骤3：编写代码
步骤4：运行验证
  → 常见问题（FAQ）

总结 + 源码链接
```

### 问题解决结构

```
问题描述（报错信息/现象）
  → 原因分析
  → 解决方案（按推荐程度排序）
  → 验证结果
  → 补充说明
```

## 语言风格

- **中性、实用主义**，不需要太多修饰
- 直接说"怎么做"，减少废话
- 步骤清晰，每步都要可复现
- 适当标注版本号和注意事项
- 语气简洁专业，像同事之间分享经验
- 避免"废话开场"：「在当今快速发展的互联网时代……」❌

## 排版规范

- 支持富文本和 Markdown 编辑器
- 代码块高亮支持好，适合贴大段代码
- 合理使用目录（TOC）功能
- 每个步骤用明确的标题标注
- 重要配置/命令用代码块包裹
- 运行结果单独展示
- 合理使用 CSDN 的折叠功能隐藏过长的输出

## 技术元素处理

### 代码
- 完整可运行的代码，标注语言类型
- 关键行加注释
- 附上完整的配置文件内容
- 运行结果截图或文本输出

### 仓库链接
- 文中直接贴链接
- 推荐附上「完整源码地址：github.com/xxx」
- 如果是自己项目，注明开源协议

### 依赖/环境
- 文章开头列出环境清单：

```markdown
## 环境说明
- 操作系统：Ubuntu 22.04
- Go 版本：1.22
- 框架版本：Gin v1.9
```

## SEO 优化

- **标题包含核心关键词**（最重要的 SEO 因素）
- 文章开头 100 字内出现关键词
- 小标题包含相关关键词
- 代码块标注语言类型（利于搜索匹配）
- 添加 3-5 个标签
- 如果是系列文章，建立专栏

## 输出模板

```markdown
# {关键词标题}

## 前言

{1-2句话说明本文解决什么问题}

## 环境说明

- 系统/语言/框架版本

## {步骤/核心内容}

### 1. {第一步}

{说明 + 代码}

### 2. {第二步}

{说明 + 代码}

## {常见问题/FAQ}（可选）

**Q: {问题1}**
A: {解决方案}

**Q: {问题2}**
A: {解决方案}

## 总结

{1-2句话总结核心要点}

完整源码：{GitHub 链接}
```
